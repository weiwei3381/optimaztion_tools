# 共生生物搜索：一种新的元启发式优化算法

## 引言

工程优化是一个具有挑战性的研究领域，近年来受到越来越多的关注。各种基于梯度的优化方法被发展用来解决各种工程优化问题。大多数使用分析或数值方法，需要梯度信息来改善初始解。然而，基于梯度的优化方法不足以解决当今许多实际工程设计问题中固有的复杂性。此外，在存在多个局部最优值的问题中，梯度搜索是困难且不稳定的[1]。目前基于梯度的工程优化方法的缺陷，鼓励研究人员开发更好的工程优化方法。

全世界在元启发式领域的研究已经产生了比传统的基于梯度的方法更优越的优化方法。Osman和Laporte将元启发式定义为一个迭代生成过程，它集成了探索和利用搜索空间的不同概念来指导从属启发式，并使用学习策略来构造信息以有效地找到接近最优解[2]。元启发式算法的例子包括：遗传算法（GA）[3]、粒子群优化（PSO）[4]、差分进化（DE）[5]、蚁群优化（ACO）[6]、和谐搜索（HS）[7]、人工蜂群（ABC）[8]、蜜蜂算法（BA）[9]、萤火虫算法[10]、充电系统搜索（CSS）[11]、[12]、Big Bang–Big Crunch（BB–BC）[13]，布谷鸟搜索（CS）[14]，矿井爆炸算法（MBA）[15]，水循环算法[16]，海豚回声定位[17]，光线优化[18]。

几乎所有的元启发式算法都有以下相同的特点：它们是受自然启发的；它们利用随机变量；它们不需要大量的梯度信息；它们有几个参数需要适合手头的问题[19]。每种元启发式算法在噪声环境、不确定参数的存在以及在不同的问题空间中的鲁棒性和性能方面都具有独特的优势[20]。然而，根据“没有免费午餐”定理，一个元启发式算法不可能最优地解决所有优化问题[21]。因此，不断需要新的高性能元启发式算法来处理特定的优化问题。

本文介绍了一种新的简单而强大的元启发式算法&共生生物搜索（Symbiotic Organisms Search, SOS）。该算法模拟了生物体在生态系统中生存的共生互动策略。与大多数其他元启发式算法相比，SOS算法的一个主要优点是算法操作不需要特定的算法参数。

论文的其余部分安排如下：第2节详细介绍了SOS算法；第3节将SOS性能与著名算法进行了比较，包括GA、DE、PSO、BA、MBA和CS；第4节讨论了SOS性能特征；第5节给出了结论。

## 共生生物搜索（SOS）算法

提出的SOS算法模拟了自然界中生物体之间的交互行为。由于依赖其他物种来维持生存甚至生存，有机体很少孤立地生活。这种依赖关系被称为共生关系。下面的小节阐明了共生的含义，给出了共生关系原型的例子，并描述了共生在生态系统中的作用。

### 共生的基本概念

共生关系来源于希腊语中“共同生活”一词。德巴里在1878年首次使用这个词来描述不同生物的同居行为[22]。今天，共生关系被用来描述任何两个不同物种之间的关系。共生关系可以是专属性的，也可以是兼性的，也可以是兼性的，这意味着两个有机体选择在互利但不重要的关系中同居。

自然界中最常见的共生关系是共生、共生和寄生。共生关系指的是两个不同物种之间的共生关系，在这种关系中双方都受益。共生关系是两种不同物种之间的共生关系，其中一种有益，另一种不受影响或中性。寄生是两种不同物种之间的共生关系，其中一种有益，另一种受到积极损害。

图1显示了一组共生生物生活在一个生态系统中。一般来说，有机体发展共生关系是为了适应环境的变化。共生关系也可能有助于生物体提高适应度和长期生存优势。因此，我们有理由得出这样的结论：共生已经并将继续塑造和维持所有现代生态系统。

### 共生生物搜索算法

当前的元启发式算法模拟自然现象。例如，人工蜂群（ABC）模拟蜂群的觅食行为，粒子群优化算法模拟动物的群集行为，遗传算法模拟自然进化过程。SOS模拟了一对有机体关系中的共生相互作用，用来寻找最适合的有机体。该算法最初是为求解连续搜索空间上的数值优化问题而开发的。

与其他基于群体的算法类似，该算法在寻找全局最优解的过程中，迭代地使用一组候选解到搜索空间中有希望的区域。SOS始于一个称为生态系统的初始种群。在初始生态系统中，随机生成一组生物到搜索空间。每个有机体代表对应问题的一个候选解决方案。生态系统中的每一个有机体都有一定的适应度值，它反映了对所期望目标的适应程度。

几乎所有的元启发式算法在每次迭代中都会对解应用一系列操作，以便为下一次迭代生成新的解。标准遗传算法有两个算子，即交叉算子和变异算子。和声搜索提出了三个规则即兴产生一个新的和声：记忆考虑，音高调整，随机选择。在ABC算法中引入了寻找最佳食物源的三个阶段。这是雇佣蜂、旁观者蜂和侦察蜂阶段。在SOS中，新溶液的生成是通过模拟生态系统中两个生物体之间的生物相互作用来控制的。介绍了与真实生物相互作用模型相似的三个阶段：互惠阶段(mutualism phase)、共生阶段(commensalism phase)和寄生阶段(parasitism phase)。

相互作用的性质决定了每个阶段的主要原理。在互惠阶段，相互作用对双方都有利；在共生阶段对一方有利，对另一方不产生影响；在寄生阶段，一方受益，另一方积极损害。每个有机体在各个阶段都随机地与另一个有机体相互作用。重复该过程，直到满足终止标准。下面的算法大纲反映了上述解释：

#### 寄生阶段

寄生的一个例子是疟原虫，它利用它与按蚊的关系在人类宿主之间传播。当这种寄生虫在人体内繁衍生息的时候，它的人体宿主会患上疟疾，并可能因此而死亡。

在SOS中，通过创造一种被称为“寄生虫向量（Parasite_Vector）”的人工寄生虫，有机体![](http://latex.codecogs.com/gif.latex?X_i)被赋予了类似于按蚊的角色。通过复制生物体![](http://latex.codecogs.com/gif.latex?X_i)在搜索空间中创建“寄生虫向量”，然后使用随机数修改随机选择的维度。![](http://latex.codecogs.com/gif.latex?X_j)是从生态系统中随机选择的一种生物，作为寄生虫的宿主。“寄生虫向量”试图在生态系统中取代![](http://latex.codecogs.com/gif.latex?X_j)。然后对这两种生物进行评估，以测量它们的适应度。如果“寄生虫向量”具有更好的适应值，它将杀死![](http://latex.codecogs.com/gif.latex?X_j)生物，并占据其在生态系统中的地位。如果![](http://latex.codecogs.com/gif.latex?X_j)的适应值更好，![](http://latex.codecogs.com/gif.latex?X_j)将具有对寄生虫的免疫力，“寄生虫向量”将无法再生活在该生态系统中。

首先对![](http://latex.codecogs.com/gif.latex?X_i)中某个或多个元素进行随机修改，得到一个新个体，叫做“寄生向量”，记作$X_{pv}$；然后从种群中随机选出一个个体$X_j(j \ne i)$作为$X_{pv}$的“宿主”，计算“寄生向量”和“宿主”的适应度值并进行比较。若“寄生向量”的适应度值更好，那么生物![](http://latex.codecogs.com/gif.latex?X_j)将会被其取代，否则![](http://latex.codecogs.com/gif.latex?X_j)将具有免疫性，被保留下来。
